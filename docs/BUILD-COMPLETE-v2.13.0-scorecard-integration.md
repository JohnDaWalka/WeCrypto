## What's New in v2.13.0

### 1. Automatic Prediction Recording
Every prediction generated by the engine is automatically captured:
```javascript
// Captured data:
{
  coin: "BTC",           // Coin symbol
  direction: "UP",       // Prediction direction (UP/DOWN/FLAT)
  confidence: 85,        // Confidence 0-100%
  signals: {...},        // Indicator signals (RSI, MACD, etc)
  timestamp: 1714520100000
}
```

**Frequency:** Every 15 minutes when predictions are generated

### 2. Automatic Settlement Recording
Every Kalshi contract outcome is automatically recorded when it settles:
```javascript
// Captured data:
{
  coin: "BTC",           // Coin symbol
  outcome: "UP",         // Actual market outcome
  modelCorrect: true,    // Was model prediction correct?
  marketCorrect: true,   // Was market prediction (Kalshi) correct?
  confidence: 85,        // Model confidence on this prediction
  timestamp: 1714520100000
}
```

**Frequency:** Every 15 minutes at bucket close (when contracts settle)

### 3. Automatic Error Tracking
All prediction failures and API errors are tracked:
```javascript
// Error types:
"NO_PREDICTION"      // Prediction engine returned nothing
"NO_PRICE_DATA"      // Prediction missing price data
"SIGNAL_DISABLED"    // Signal temporarily disabled (awaiting feed)
"API_ERROR"          // Kalshi API error
"CONNECTION_ERROR"   // Network error
```

### 4. Auto-Correlation
Predictions are automatically matched to settlements by:
- Coin symbol
- Time window (1-hour match window)
- Creates linked accuracy pairs

### 5. Console Commands
Access prediction accuracy directly from browser console:

```javascript
// View full accuracy report
window._aggregator.printReport()

// Get accuracy by coin
window._aggregator.getAccuracy()

// View recent errors
window._aggregator.getRecentErrors()

// Export to CSV
const csv = window._aggregator.exportCSV()
navigator.clipboard.writeText(csv)

// Run 5-stage health diagnostics
await window._aggregator.diagnoseSettlementData()

// View all predictions
window._aggregator.getPredictions()

// View all settlements
window._aggregator.getSettlements()
```

## Troubleshooting

### Issue: "Aggregator not defined" in console
**Cause:** Modules didn't load  
**Fix:** Check that index.html has script tags:
```html
<script src="src/kalshi/scorecard-data-aggregator.js"></script>
<script src="src/kalshi/accuracy-scorecard-comprehensive.js"></script>
```

### Issue: No predictions appearing
**Cause:** Prediction engine not running or disabled  
**Fix:** 
1. Check predictions are being generated: `window._predictions`
2. Check if prediction for coin is disabled: `SIGNAL_DISABLED_COINS`
3. Restart app

### Issue: Settlements not matching
**Cause:** Timing mismatch between prediction and settlement  
**Fix:**
1. Allow 1 hour window for matching
2. Check coin symbols match exactly
3. Verify settlements are actually resolving

### Issue: Memory usage growing
**Cause:** Normal - circular buffers keep last 24-48h of data  
**Status:** This is expected, not a bug
