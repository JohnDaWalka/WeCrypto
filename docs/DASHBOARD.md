# 📊 Dashboard Guide

How to read and use the WE-CRYPTO prediction dashboard.

---

## Overview

The dashboard is a real-time view of predictions, accuracy, Kalshi market data, and trade intents for 7 coins: **BTC, ETH, SOL, XRP, DOGE, BNB, HYPE**.

---

## Main Panels

### 1. Predictions Panel

Displays the current UP/DOWN prediction for each coin.

| Column | Description |
|--------|-------------|
| **Coin** | Ticker symbol |
| **Direction** | `UP` or `DOWN` |
| **Confidence** | 0–100% — model certainty |
| **Score** | Raw model score (-1.0 to +1.0) |
| **Horizon** | 5M / 15M / 1H window |

**How to read confidence:**
- **>65%** — Strong signal, model has high conviction
- **50–65%** — Moderate signal, proceed with caution
- **<50%** — Weak/noisy signal, not recommended

---

### 2. Accuracy Scorecard

Live per-signal accuracy tracking updated every 30 seconds.

| Column | Description |
|--------|-------------|
| **Signal** | Indicator name (RSI, MACD, etc.) |
| **Accuracy** | % of correct calls over last N contracts |
| **Count** | Number of settled contracts evaluated |
| **Trend** | ↑ improving / ↓ declining / → stable |
| **Weight** | Current adaptive weight multiplier |

**Status indicators:**
- 🟢 **52%+** — Signal is performing well, weight is boosted
- 🟡 **47–52%** — Neutral, weight unchanged
- 🔴 **<47%** — Underperforming, weight is reduced

---

### 3. Kalshi Panel

Shows live Kalshi binary contract data.

| Column | Description |
|--------|-------------|
| **Contract** | Kalshi contract ID |
| **Strike** | Price target (floor_strike) |
| **YES %** | Current YES price (implied probability) |
| **Direction** | `above` or `below` |
| **EV** | Expected value vs. model probability |
| **Close** | Time until contract settles |

**EV interpretation:**
- **EV > 3** — Model sees significant edge over Kalshi pricing
- **EV 0–3** — Small edge, lower confidence
- **EV < 0** — Model disagrees with Kalshi direction

---

### 4. Orchestrator / Trade Intents

Trade intents generated by `floating-orchestrator.js`.

| Column | Description |
|--------|-------------|
| **Coin** | Ticker |
| **Intent** | `BUY YES`, `BUY NO`, or `WAIT` |
| **Edge** | EV in cents per dollar risked |
| **Kelly** | Suggested position size (Kelly fraction, capped 25%) |
| **Reason** | Which signals triggered the intent |

---

## Status Indicators

| Indicator | Meaning |
|-----------|---------|
| 🟢 **LIVE** | Data feed active and fresh |
| 🟡 **STALE** | Data >90 seconds old |
| 🔴 **OFFLINE** | Feed disconnected |
| ⚙️ **TUNING** | Weights being updated |
| 📚 **LEARNING** | Accumulating initial data (first 2 min) |

---

## Real-Time Updates

The dashboard refreshes on a **30-second cycle**:

```
0s:  Fetch OHLCV candles (Coinbase → Binance fallback)
5s:  Fetch Kalshi 15M + 5M markets
10s: Fetch Polymarket data
15s: Run prediction model
20s: Calculate signal accuracy
25s: Auto-tune weights (if 2 min elapsed)
30s: Render updated dashboard
```

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `F5` | Force refresh all data |
| `F12` | Open DevTools console |
| `Ctrl+Shift+D` | Toggle debug panel |

---

## Console Commands

Open DevTools (F12) for additional diagnostics:

```js
// Force a full prediction refresh
window.PredictionEngine?.refresh()

// Show current weights
window.AdaptiveLearningEngine?.getWeights()

// Show Kalshi data for a coin
window.PredictionMarkets?.getCoin('BTC')
```
