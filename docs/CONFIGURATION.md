## Environment Variables

Copy `.env.example` to `.env` and fill in values before running.

### API Credentials

| Variable | Required | Description |
|----------|----------|-------------|
| `KALSHI_API_KEY` | Yes | Kalshi exchange API key |
| `KALSHI_API_SECRET` | Yes | Kalshi exchange API secret |
| `POLYMARKET_API_KEY` | No | Polymarket API key (optional) |
| `COINBASE_API_KEY` | No | Coinbase Advanced API key (for higher rate limits) |
| `COINBASE_API_SECRET` | No | Coinbase Advanced API secret |

### Proxy & Network

| Variable | Default | Description |
|----------|---------|-------------|
| `PROXY_PORT` | `3010` | Starting port for the local Rust proxy |
| `KALSHI_WORKER_PORT` | `3050` | Port for the Kalshi Node.js worker |
| `WEB_SERVICE_PORT` | `3443` | Optional web mirror service port |

### Feature Flags

| Variable | Default | Description |
|----------|---------|-------------|
| `ENABLE_PYTH` | `false` | Enable Pyth Network oracle integration |
| `ENABLE_WEB_SERVICE` | `false` | Enable web mirror service |
| `ENABLE_LLM_LAYER` | `false` | Enable LLM-enhanced signal processing |
| `DEBUG_MODE` | `false` | Enable verbose console logging |

## Tuning Parameters

These control the adaptive learning engine. Most users should not need to change defaults.

### Weight Adjustment

| Parameter | Default | Description |
|-----------|---------|-------------|
| `BOOST_THRESHOLD` | `0.52` | Accuracy % above which a signal weight is boosted |
| `REDUCE_THRESHOLD` | `0.45` | Accuracy % below which a signal weight is reduced |
| `BOOST_AMOUNT` | `0.05` | Amount to add to weight per tuning cycle |
| `REDUCE_AMOUNT` | `0.05` | Amount to subtract from weight per tuning cycle |
| `MAX_WEIGHT` | `2.0` | Maximum weight cap for any signal |
| `MIN_WEIGHT` | `0.1` | Minimum weight floor for any signal |
| `TREND_BOOST_AMOUNT` | `0.08` | Boost/reduce amount when a trend is detected |

### Tuning Cycle Timing

| Parameter | Default | Description |
|-----------|---------|-------------|
| `POLLING_INTERVAL_MS` | `30000` | Main prediction cycle (30 seconds) |
| `TUNING_INTERVAL_MS` | `120000` | Weight update frequency (2 minutes) |
| `HISTORY_WINDOW_HOURS` | `5` | Hours of settled contracts to evaluate |
| `MIN_CONTRACTS_REQUIRED` | `10` | Minimum settled contracts before tuning |

## Gate Thresholds

These control when a trade intent is generated by the orchestrator.

| Parameter | Default | Description |
|-----------|---------|-------------|
| `MIN_EV_CENTS` | `3` | Minimum expected value (cents) to generate intent |
| `MIN_CONFIDENCE` | `0.55` | Minimum model confidence (55%) |
| `MAX_KELLY_FRACTION` | `0.25` | Maximum Kelly fraction (25%) |
| `MIN_KALSHI_LIQUIDITY` | `100` | Minimum open interest on Kalshi contract |

## Signal Weights (Initial)

Default starting weights before adaptive tuning takes effect. Stored in `beta1_weights` in localStorage.

| Signal | Default Weight |
|--------|---------------|
| RSI | 1.00 |
| MACD | 1.00 |
| CCI | 1.00 |
| Fisher Transform | 1.00 |
| ADX | 1.00 |
| ATR | 1.00 |
| Order Book Imbalance | 1.00 |
| Kalshi Probability | 1.00 |
| Crowd Fade | 1.00 |

Weights are automatically adjusted over time. To reset to defaults:

```js
localStorage.removeItem('beta1_weights')
location.reload()
```

The canonical coin universe is fixed across all modules. Adding or removing coins requires synchronised changes in predictions, Kalshi mapping, orchestrator, and UI tables.

```
BTC, ETH, SOL, XRP, DOGE, BNB, HYPE
```
