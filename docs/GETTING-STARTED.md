# 🚀 Getting Started with WE-CRYPTO

Complete guide to installing and running the WE-CRYPTO self-teaching prediction engine.

---

## Prerequisites

- **Node.js** v18+ (v20 LTS recommended)
- **npm** or **pnpm** v8+
- **Windows 10/11** (for the Electron desktop app)
- API credentials (see [Configuration](./CONFIGURATION.md))

---

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/JohnDaWalka/WE-CFM-Orchestrator.git
cd WE-CFM-Orchestrator
```

### 2. Install Dependencies

```bash
npm install
# or
pnpm install
```

### 3. Configure Environment

```bash
cp .env.example .env
# Open .env in your editor and fill in API credentials
```

See [CONFIGURATION.md](./CONFIGURATION.md) for all available variables.

---

## Running the App

### Development Mode

```bash
npm start
# or
pnpm run dev
```

This starts the Electron desktop app with DevTools enabled.

### Production Build

```bash
# Portable single-file executable
npm run build:portable

# Windows installer
npm run build:installer
```

Output goes to `dist/`. See [DEPLOYMENT.md](./DEPLOYMENT.md) for more.

## First Run Walkthrough

After launching the app:

1. **0–30 sec** — App boots, polling begins, Kalshi worker starts on port 3050
2. **30–60 sec** — Historical settled contracts are fetched from Kalshi/Polymarket
3. **60–120 sec** — Accuracy scorecard populates with initial signal data
4. **120 sec+** — Auto-tuning activates; weights begin adjusting every 2 minutes
5. **5–10 min** — Predictions stabilise as the learning engine accumulates data

### What You'll See

| Panel | Description |
|-------|-------------|
| **Predictions** | UP/DOWN calls with confidence % for BTC, ETH, SOL, XRP, DOGE, BNB, HYPE |
| **Scorecard** | Per-signal accuracy history and trend arrows |
| **Kalshi** | Live Kalshi contract prices and EV calculations |
| **Orchestrator** | Trade intents generated by the floating orchestrator |

## Verifying It Works

Open the DevTools console and run:

```js
// Check prediction engine is running
window.PredictionEngine

// Check latest prediction for BTC
window._predictions?.BTC

// Check adaptive weights
window.AdaptiveLearningEngine?.getWeights?.()

// Check Kalshi data
window.PredictionMarkets?.getCoin?.('BTC')
```

## Troubleshooting First-Run Issues

| Problem | Solution |
|---------|----------|
| App won't start | Ensure Node.js 18+ is installed |
| No Kalshi data | Check `.env` for valid Kalshi credentials |
| Blank predictions | Wait 2–3 minutes for data to load |
| Proxy errors | Check that port 3010 is available |

See [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) for more.
