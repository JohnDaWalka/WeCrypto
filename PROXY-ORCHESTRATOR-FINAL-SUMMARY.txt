IMPLEMENTATION DETAILS
================================================================

RATE LIMITER (Per-Endpoint)
──────────────────────────
• Tracks consecutive failures (429/503 responses)
• Exponential backoff: 2s → 4s → 8s → 16s → 32s
• Circuit breaker: Opens after 3 consecutive failures
• Circuit recovery: Automatic after 60 seconds
• Failure window: 60-second history for context
• Safe interval calculation based on endpoint config

BACKOFF SEQUENCE:
  Attempt 1: Failure → 2s backoff
  Attempt 2: Failure → 4s backoff
  Attempt 3: Failure → 8s backoff
  After 3: Circuit breaker OPEN for 60s
  Recovery: Auto-recovery after 60s, reset to attempt 1

REQUEST BATCHER (Deduplication)
──────────────────────────────
• Hash key: SHA1(URL + JSON.stringify(params))
• Window: 500ms coalescing window
• Storage: Map<key, { promise, createdAt }>
• Cleanup: Automatic after window expires
• Result: Same promise returned to all subscribers

DEDUPLICATION EXAMPLE:
  T=0ms:   Call fetch(url) → creates promise P1
  T=50ms:  Call fetch(url) → returns P1 (no new request)
  T=150ms: Call fetch(url) → returns P1 (no new request)
  T=600ms: Call fetch(url) → creates new promise P2 (window expired)

FALLBACK ROUTER (Chain Management)
──────────────────────────────────
• Sources: registrable endpoints with health tracking
• Health checks: Every 30 seconds
• Chain execution: Try each endpoint sequentially
• Skip unhealthy: Never tries circuit-broken endpoints
• Success/failure tracking per source
• Configurable fallback chains per endpoint

FALLBACK CHAIN EXAMPLE (Kalshi Markets):
  Primary: Kalshi API (https://api.elections.kalshi.com/...)
  Fallback 1: Polymarket API (https://gamma-api.polymarket.com/...)
  Fallback 2: Cache (localStorage + in-memory)
  Final: Error if all exhausted

CACHE ORCHESTRATOR (Multi-Layer)
────────────────────────────────
L1 Cache: In-Memory Promises
• TTL: 1-30 seconds (per cache type)
• Storage: Map<key, { value, expiresAt }>
• Access: < 1ms
• Size: ~5-20MB typical

L2 Cache: localStorage
• TTL: 5 minutes standard
• Storage: localStorage (po_cache_* prefix)
• Access: < 10ms
• Size: Limited by browser (5-10MB)

L3 Cache: Disk (Optional)
• Via Tauri bridge (F:\WECRYP\data)
• Not currently active (future enhancement)

L4 Layer: Network
• Final fetch with all resilience
• Auto-caching of successful results

CACHE TTL BY TYPE:
  price-quotes: 5s (prices change rapidly)
  market-data: 30s (metadata stable)
  settlement: 1h (immutable after settled)
  fear-greed: 1h
  global-metrics: 5min

PROXY ORCHESTRATOR (Main Interface)
───────────────────────────────────
Main Method:
  async fetch(url, options = {})
    - Unified entry point for all API calls
    - Orchestrates cache → dedupe → rate limit → fetch → fallback
    - Returns Promise<any>
    - Automatically caches successful results

Health Diagnostics:
  getHealthStatus() → {
    uptime: ms,
    endpoints: { kalshi: {...}, cmc: {...}, ... },
    cache: { l1Size, hitRate, hitStats },
    requests: { total, byEndpoint },
    failures: { total, byEndpoint },
    latency: { average, min, max },
    batcher: { activeBatches, windowMs },
    fallback: { sources: {...} }
  }

Metrics Export:
  • Collected every 30 seconds
  • Logged to console [ProxyOrchestrator] prefix
  • Can export via window.tauriExportMetrics()
  • JSON Lines format: one metric per line
